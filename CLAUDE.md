# CLAUDE.md

Chrome Manifest V3 extension. Two modes, gated by the active tab's URL:

- **YouTube mode** — on `www.youtube.com/watch?v=…` the popup extracts the transcript into a searchable preview pane with clickable timestamps, and lets the user copy to clipboard or download as TXT / SRT.
- **Page mode** — on any other (non-restricted) page the popup hides the transcript UI and offers a single Summarise action that scrapes the page's main text and runs it through the configured LLM. No download, no transcript pane, no Copy-transcript button.

**Intended for the Chrome Web Store** — avoid anything that looks copy-pasted from a competitor.

## Strict scope rules

Do not break these without an explicit instruction:

- **Manifest V3 only.**
- **Permissions:** `activeTab`, `scripting`, `storage`. **`host_permissions`** are limited to the six supported LLM providers + Notion: `api.deepseek.com`, `api.openai.com`, `api.anthropic.com`, `openrouter.ai`, `api.groq.com`, `generativelanguage.googleapis.com`, `api.notion.com`. Do not add others without explicit instruction. No remote code, no external fonts, no third-party SDKs.
- **Silver-toned icons** (`icon16.png`, `icon48.png`, `icon128.png`) generated by `/tmp/gen_icons.py` (Pillow). Registered under both `icons` and `action.default_icon`. If the design needs tweaking, regenerate from that script rather than editing PNGs by hand.
- **No background script / service worker.** The keyboard shortcut uses the built-in `_execute_action` command. If the popup doesn't open on the shortcut, the user has almost certainly not rebound it at `chrome://extensions/shortcuts` — Chrome drops the binding when the command name or manifest changes. Do **not** attempt workarounds with `chrome.action.openPopup()`, `chrome.windows.create()`, or any on-page banner injection; all have been tried and explicitly rejected.
- **No content script file, no on-page banner.** Transcript extraction and video seeks both go through `chrome.scripting.executeScript` from the popup. All UI lives inside the popup window.
- **No auto-copy, no auto-summarise.** Extraction runs automatically on popup open to populate the preview, but the clipboard write, every LLM call, and the Obsidian/Notion saves only happen on explicit button clicks.
- **AI summary:** BYOK only — user supplies their own API key for one of six providers (DeepSeek, OpenAI, Anthropic, OpenRouter, Groq, Gemini), stored in `chrome.storage.local`. No shipped default key, no on-device LLM. Save targets are **Obsidian** (via `obsidian://adv-uri` — needs the Advanced URI plugin) and **Notion** (via `api.notion.com` with a user-supplied integration token + page ID).
- **No TypeScript, no bundler, no npm, no `package.json`, no `node_modules`, no `.gitignore`, no license.**
- **File set:** `manifest.json`, `popup.html`, `popup.js`, `README.md`, `icon{16,48,128}.png`, and this `CLAUDE.md`. Do not add more without instruction.

## How extraction works

`popup.js` injects `scrapePage` into the active tab via `chrome.scripting.executeScript({ world: 'MAIN', func: ... })`. The injected function:

1. Waits for `ytd-watch-flexy[video-id]` to match the URL's `?v=` parameter, so SPA navigations don't return stale data from the previous video.
2. Pulls the player response: `document.getElementById('movie_player').getPlayerResponse()`, falling back to `window.ytInitialPlayerResponse`.
3. Picks a caption track from `response.captions.playerCaptionsTracklistRenderer.captionTracks` — preference: manual (no `kind:'asr'`) + English, then manual any-lang, then English ASR, then first available.
4. Fetches `<track.baseUrl>&fmt=json3` (same-origin to youtube.com from MAIN world — no host_permission needed for this fetch). Parses the `events[]` array into `{timestamp, text}` segments.
5. Pulls metadata from `response.videoDetails` (title, channel, lengthSeconds) and `response.microformat.playerMicroformatRenderer.publishDate`.

Crucially, this **never touches the transcript engagement panel**. No DOM clicks, no panel toggling, no scroll movement, no layout shift — even in theatre mode. Returns `{ segments: [{timestamp,text}], info: {...} }` on success, or `{ error: code, debug? }` on failure. Error codes:
- `no-button` — player response missing, or video has no caption tracks.
- `panel-empty` — caption URL fetched but yielded no segments.

The function must stay self-contained — it's serialised and run in the page, not the popup.

## How page-mode extraction works

`initPageMode()` runs when the active tab isn't a YouTube watch page and the URL scheme is one Chrome lets extensions read (excludes `chrome:`, `chrome-extension:`, `edge:`, `about:`, `view-source:`, `devtools:`, plus the Chrome Web Store hosts). It hides the Copy-transcript button, leaves the transcript `<details>` collapsed, and sets the status to "Ready to summarise this page."

On Summarise click, `readPageText()` injects a one-shot scraper via `chrome.scripting.executeScript` (default ISOLATED world is fine — no MAIN-world access needed). The scraper picks the first matching root from `article` / `main` / `[role=main]` / `body`, reads `innerText` (so CSS-hidden cookie banners and nav menus drop out), collapses runs of whitespace, and returns `{ title, text }`. The LLM call uses `PAGE_SUMMARY_SYSTEM_PROMPT` (parallel to the YouTube one) and reuses every downstream flow — bullet rendering, Copy, Save to Obsidian, Save to Notion. The page URL fills the same role the YouTube URL fills in transcript mode.

## How seeking works

Clicking a timestamp chip in the preview injects a two-line function into the tab that sets `document.querySelector('video').currentTime` and focuses the tab. Uses `chrome.scripting.executeScript` with `args` to pass the target time.

## Popup UI

Layout (top to bottom):

1. **Header** — icon + product name + scraped video title (truncated).
2. **Status line** — loading / error / success.
3. **Actions block** (`#controls`, hidden until extraction succeeds):
   - **Copy transcript** — primary button (muted slate).
   - **Summarise** — secondary button (muted forest green).
   - **Extra focus** input — persisted, woven into the system prompt; pressing Enter triggers Summarise.
   - **Summary section** (hidden until populated) — bullet points + Copy / Save to Obsidian / Save to Notion buttons.
4. **Transcript** disclosure (`<details>`, collapsed by default; auto-collapses again after a successful Summarise) — search input, 160px preview list with clickable timestamp chips, and `transcript-controls` divider holding the **Include timestamps** + **Strip [Music]** toggles and **TXT / SRT** download buttons.
5. **Settings** disclosure (always visible) — three `settings-section` blocks (AI provider, Save to Obsidian, Save to Notion) plus a Keyboard shortcut row and a hint footer.

TXT and preview share the same formatter (`buildText` → `effectiveSegments`), so what you see is what gets copied. SRT is standard subtitle format with one cue per segment. Both formats append a metadata footer identifying the source.

## How summary + save work

- `summarise()` reads the persisted `llmProvider` / `llmModel` / `llmKey` and dispatches to `callLLM`, which routes by provider family:
  - **`openai`** family (DeepSeek, OpenAI, OpenRouter, Groq) → `POST <baseUrl>/chat/completions` with `Authorization: Bearer <key>`.
  - **`anthropic`** family (Claude) → `POST /v1/messages` with `x-api-key`, `anthropic-version: 2023-06-01`, and `anthropic-dangerous-direct-browser-access: true`.
  - **`gemini`** family → `POST /v1beta/models/<model>:generateContent?key=<key>` with `system_instruction` + `contents`.
- Transcript text is capped at 80k chars (`SUMMARY_INPUT_CAP`). System prompt asks for 5–7 plain bullets — no verdict, no rating, no headings.
- Bullets are parsed by `parseSummaryResponse`, which strips bullet markers / numbering so the renderer applies its own list styling.
- The shared markdown payload is `## <title>\n<youtube url>\n\n- bullet…` — used for both clipboard copy and Obsidian save.
- Obsidian: opens `obsidian://adv-uri?vault=…&filepath=…&data=…&mode=append` via `chrome.tabs.create`. **Requires the Advanced URI community plugin** in the user's vault.
- Notion: `PATCH https://api.notion.com/v1/blocks/<page_id>/children` with one `heading_2` (title) + `paragraph` (link) + N `bulleted_list_item` blocks. Page ID is normalised (dashes stripped). Notion-Version header pinned to `2022-06-28`.
- One-time migration on first load: any prior `deepseekKey` storage value is moved to `llmKey` with `llmProvider: 'deepseek'`.

## Theme

DaisyUI **business** palette ported as plain CSS variables — **low-key variant** with desaturated, slightly cooler hues so nothing shouts. Dark by default with a `prefers-color-scheme: light` variant that lifts the base layers. Primary action button = muted slate `#6E7E89`; AI/secondary action = muted forest `#6A9A75`. Tone palette: muted navy primary `#2D5278`, muted terracotta accent `#C66B4D`, muted brick error, muted amber warning.

## Chrome Web Store publishing

See the "Pre-publish checklist" in `README.md` for the full list. Privacy story has **two tiers**:

- **Default (no AI settings configured):** zero network requests. Transcripts stay in the browser, prefs + credentials are local-only storage.
- **With Summarise / Save configured:** transcript text is sent to the configured LLM provider host only on Summarise click; summary text is sent to `api.notion.com` only on Save to Notion click. Obsidian save uses the local `obsidian://` URL scheme — never hits the network.

The store listing must be honest about both tiers — frame the AI features as opt-in BYOK so the privacy posture stays a differentiator.
